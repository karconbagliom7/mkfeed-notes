# mkfeed-notes

这是一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要存放一系列独立的 HTML 页面，这些页面可能包含笔记、工具、归档内容或其他静态信息。每个页面都是自包含的，不依赖特定域名或外部网站，便于本地浏览或通过静态托管服务发布。

## 目录说明

```
mkfeed-notes/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的资源文件（如样式、脚本、图片等）
├── index.html      # 可选的首页，用于列出或导航到各页面
└── README.md       # 本文件
```

- **pages/**：每个 HTML 页面独立存放，命名清晰，便于查找。
- **assets/**：公共资源文件夹，避免页面间重复引用。
- **index.html**：可按需创建，用于提供页面索引或简单导航。

## 页面归档说明

- 每个 HTML 页面均为独立文件，可直接在浏览器中打开使用。
- 页面内容以实用信息为主，不包含广告、跟踪脚本或外部依赖。
- 归档内容可能随时间更新、补充或清理，以保持仓库整洁。

## 维护说明

- 欢迎通过 Issue 或 Pull Request 提交建议或修正。
- 新增页面时，请确保文件命名直观，并尽量放置在 `pages/` 目录下。
- 资源文件统一存放在 `assets/` 中，避免散落各处。
- 本仓库不提供具体网站或服务的链接，仅作为静态页面存档。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件。
3. 或将整个仓库部署到任何静态托管服务，即可在线访问。
